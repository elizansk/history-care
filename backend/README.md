# Backend для History Care 

## Краткое описание проекта

Backend часть реализована на языке Go и предоставляет REST API для управления данными о исторических зданиях и сбора средств на их реконтрукцию. Включает модели для пользователей, зданий, регионов, категорий, ресурсов и заказов на реконструкцию. Использует Gin для HTTP сервера, GORM для работы с PostgreSQL и godotenv для загрузки переменных окружения.

## Описание файловой структуры проекта

```
backend/
├── go.mod                    # Файл зависимостей Go
├── go.sum                    # Контрольные суммы зависимостей
├── cmd/
│   └── awesomeProject/
│       └── main.go           # Точка входа в приложение
└── internal/
    ├── api/
    │   └── server.go         # Настройка сервера Gin, маршруты API
    ├── app/
    │   ├── handler/
    │   │   └── handler.go    # HTTP обработчики для маршрутов
    │   └── repository/
    │       └── repository.go # Репозиторий для взаимодействия с БД
    ├── models/
    │   └── models.go         # Определения моделей данных
                
```

## Инструкция по установке и запуску

### Предварительные требования

- Go 1.25.0 или выше установлен.
- PostgreSQL база данных доступна.
- Переменные окружения настроены (см. ниже).

### Шаги установки

1. **Установите зависимости:**
   ```bash
   go mod download
   ```

2. **Настройте базу данных:**
   Создайте базу данных PostgreSQL и настройте переменные окружения.

3. **Запустите приложение:**
   ```bash
   go run cmd/awesomeProject/main.go
   ```

   Сервер запустится на порту 8080.


## Список используемых технологий

- **Go 1.25.0** — язык программирования. [Документация](https://golang.org/doc/)
- **Gin** — веб-фреймворк для Go.[Документация](https://gin-gonic.com/ru/docs/)
- **GORM** — ORM для Go. [Документация](https://gorm.io/)
- **PostgreSQL** — реляционная база данных. [Документация](https://www.postgresql.org/docs/)
- **godotenv** — библиотека для загрузки переменных окружения из .env файла. [Документация](https://github.com/joho/godotenv)

## Описание переменных окружения

- `DB_HOST` — хост базы данных PostgreSQL
- `DB_PORT` — порт базы данных
- `DB_USER` — имя пользователя БД
- `DB_PASSWORD` — пароль пользователя БД
- `DB_NAME` — имя базы данных
- `TEMPLATES_PATH` — путь к HTML шаблонам
- `STATIC_PATH` — путь к статическим файлам

## Описание неочевидных (временных) технических решений 

- **Многослойная архитектура:** Разделение на models, repository и handler обеспечивает разделение ответственности и упрощает тестирование.
- **Загрузка шаблонов и статических файлов:** Сервер Gin загружает HTML шаблоны и статические ресурсы из указанных путей, что позволяет рендерить страницы на сервере.
- **Поддержка нескольких окружений:** В зависимости от GO_ENV загружается соответствующий .env файл.
